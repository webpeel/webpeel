Web Scraping Benchmarks

We tested WebPeel, Firecrawl, Tavily, and a raw fetch baseline across 30 diverse URLs — measuring success rate, latency, content quality, and extraction depth. Here's what we found.

Building a web scraping tool for AI agents means competing with well-funded, established players. Rather than make claims we can't back up, we decided to run an honest benchmark. We tested every tool under the same conditions, on the same URLs, and we're publishing the raw data so anyone can verify.

A note on honesty

We're publishing results where we lose, too. We think that's more useful than cherry-picking wins. If a competitor does something better, we want to know — and we want you to know. Our benchmark code is open source.

Methodology

How we tested

  • 30 URLs across 6 difficulty tiers: static HTML, dynamic content, JS-heavy SPAs, protected/anti-bot sites, documents, and international/edge cases
  • Serial execution — one URL at a time, no parallelism, to get fair timing
  • 30-second timeout per URL
  • 3 runs per tool — we report the median
  • Same machine — Mac Mini M2, macOS Sequoia, gigabit fiber
  • Default settings — no special config, just the URL
  • WebPeel runs locally (CLI); competitors use their hosted APIs

Why WebPeel is slower (and why we accept it)

WebPeel runs locally and starts with a lightweight fetch, but it aggressively escalates when needed (browser rendering, stealth, anti-bot strategies). That extra work increases median latency, yet it is exactly why WebPeel finished at 30/30 with zero failures while others dropped URLs in protected tiers.

Key Failures Observed

Instead of cherry-picking, here are the exact failure patterns we saw in the 30-URL run:

  • Firecrawl — explicitly blocks LinkedIn and Reddit with "We do not support this site."
  • Tavily — failed on example.com, Medium, Bloomberg, Glassdoor, and SEC EDGAR.
  • Raw Fetch — failed on targets behind Cloudflare/bot detection.
  • WebPeel v3 — 100% success (30/30), zero failures.

Where We Win

  • Reliability — WebPeel was the only runner to reach 30/30 (100%) across all six tiers.
  • Protected targets — WebPeel went 5/5 on protected sites, including cases where other tools blocked or failed.
  • Content quality — highest average quality score in the run (0.870).
  • Local-first architecture — WebPeel runs locally (no cloud API round-trip), with smart escalation only when needed.
  • Open source — Fully MIT. Self-host for unlimited usage with no vendor lock-in.

Conclusion

For this benchmark run, WebPeel achieved a clean 30/30 (100%) across all 30 URLs and all six tiers.

  • Use WebPeel if reliability is your top priority and you want local-first scraping with smart escalation.
  • Use Firecrawl if you already depend on their managed API ecosystem, but note the explicit LinkedIn/Reddit blocks in this run.
  • Use Tavily if raw latency is your top metric and you can accept lower coverage on protected/document targets.
  • Use Raw Fetch as a minimal baseline for simple pages, not as a production strategy for bot-protected targets.

We'll keep re-running this benchmark as we ship improvements. We care more about publishable truth than flattering snapshots.

Run the benchmark yourself

Our benchmark suite is open source. Clone the repo, add your API keys, and verify our results: git clone https://github.com/webpeel/webpeel && cd webpeel && npm run benchmark
