Web Scraping Benchmarks

We tested WebPeel, Firecrawl, Tavily, and Exa across 30 diverse URLs — measuring speed, success rate, and content quality. Here's what we found.

Building a web scraping tool for AI agents means competing with well-funded, established players. Rather than make claims we can't back up, we decided to run an honest benchmark. We tested every tool under the same conditions, on the same URLs, and we're publishing the raw data so anyone can verify.

A note on honesty

We're publishing results where we lose, too. We think that's more useful than cherry-picking wins. If a competitor does something better, we want to know — and we want you to know. Our benchmark code is open source.

Methodology

How we tested

  • 30 URLs across 6 difficulty tiers: static HTML, dynamic content, JS-heavy SPAs, protected/anti-bot sites, documents, and international/edge cases
  • Serial execution — one URL at a time, no parallelism, to get fair timing
  • 30-second timeout per URL
  • 3 runs per tool — we report the median
  • Same machine — Mac Mini M2, macOS Sequoia, gigabit fiber
  • Default settings — no special config, just the URL
  • WebPeel runs locally (CLI); competitors use their hosted APIs

Why WebPeel is fast

WebPeel's smart escalation tries a simple HTTP fetch first (~150ms). Only when that fails does it launch a browser (~2s) or stealth mode (~5s). For the 80% of pages that don't need JavaScript rendering, this means near-instant results. Competitors that always use a browser pay the overhead on every request.

Where We Lose

Honesty matters. Here's where competitors beat us:

  • Protected sites (Firecrawl wins) — Firecrawl's proprietary "Fire-engine" has enterprise-grade proxy rotation and anti-bot infrastructure. On heavily protected sites like Bloomberg and Glassdoor, they succeed where we don't. We're working on this.
  • Search quality (Exa wins) — Exa has their own neural search index. Our search wraps DuckDuckGo — functional, free, but not in the same league for semantic relevance. If search is your primary use case, Exa is better.
  • Scale reliability (Tavily wins) — Tavily handles 100M+ monthly requests with 99.99% SLA. We haven't been tested at that scale yet.

Where We Win

  • Speed on standard pages — Smart escalation means we skip the browser for ~80% of requests. 150-400ms median vs 500-1000ms for competitors.
  • JS-heavy pages — Auto-escalation to Playwright catches content that Tavily and Exa miss entirely (they don't render JS).
  • Documents — Built-in PDF and DOCX parsing means arXiv papers and SEC filings just work.
  • Price — $9/mo for 1,250 requests vs Firecrawl's $16/mo for 3,000 credits. But we don't feature-gate — free users get stealth, actions, everything.
  • Open source — Fully MIT. Self-host for unlimited, free usage. No vendor lock-in.

Conclusion

No tool wins everything. Here's our honest recommendation:

  • Use WebPeel if you want fast, local-first web scraping with auto-escalation, open source licensing, and the lowest cost per fetch. Best for AI agents, MCP integrations, and developers who want control.
  • Use Firecrawl if you need enterprise-grade proxy infrastructure for heavily protected sites. They have better anti-bot at scale.
  • Use Exa if semantic web search is your primary use case. Their neural index is genuinely different technology.
  • Use Tavily if you need search+extract at massive scale (100M+ requests/month) with an SLA.

We'll keep improving. The areas where we lose are our roadmap. And we'll re-run this benchmark quarterly as we ship improvements.
